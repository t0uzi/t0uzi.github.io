    宽字节注入

    原理：

    后端与mysql编码不同

    编码

    (GBK)

    当前端对我们的字符进行url编码并发送到后端时,后端对我们输入的字符进行\转义,

    后端会将转移后的数据发送到数据库,如果我们在\前加一个编码%df这样的话,当后端将加上%df后的编码发送到数据库进行操作时,若数据库为gb2312编码,那么数据库会将前两个编码识别为一个字符運,这样一来,转义的字符就会被吞掉从而实现绕过,进而注入成功

    具体流程

    用户输入 ==> 过滤函数 ==> 代码层的 $sql ==> MySQL 处理请求 ==> MySQL 结果
    %df%27 ==> addslashes ==> %df%5c%27 ==> (GBK) ==> 運
